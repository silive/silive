# TODO 与已知问题

## 1. 高优先级

### 1.1 生产环境必须重新生成强后台密码

当前交接包不包含真实后台密码。生产环境必须使用强密码：

- 管理员账号不要用 `admin`。
- 密码至少 16 位，包含大小写字母、数字和符号。
- 修改 `.env` 后重启 PM2。

### 1.2 上传目录如果由 Nginx 直接托管，需要额外禁止脚本执行

Node 层已经禁止访问脚本扩展，并写入 `.htaccess`。但 Nginx 不读取 `.htaccess`。

如果生产 Nginx 直接服务 `/uploads/`，必须添加：

```nginx
location ~* ^/uploads/.*\.(php|phtml|phar|cgi|pl|py|sh|js|html|htm|svg|xml)$ {
    return 403;
}
```

### 1.3 支付证书、公钥、APIv3 Key 需要上线前最终核验

必须确认：

- 商户号正确。
- 小程序 AppID 与商户绑定。
- 商户私钥路径正确。
- 微信支付公钥 ID 与公钥文件匹配。
- `WECHAT_PAY_NOTIFY_URL` 外网可访问。
- 生产 `PAY_MOCK=false`。

## 2. 中优先级

### 2.1 动态换肤前端暂时关闭

`utils/theme.js` 中：

```js
const ENABLE_REMOTE_THEME = false
```

后台皮肤管理仍保留，但小程序前端固定使用本地 `skin01`。

原因：此前动态换肤联动调试导致前端稳定性风险，已暂时止损。

后续建议：

- 新建分支重做动态换肤。
- 减少页面内联 style 分散逻辑。
- 用统一主题 provider 或页面 mixin 管理。
- 增加小程序真机验证。

### 2.2 JSON 兜底存储不适合生产并发

如果 `mysql2` 未安装或 MySQL 不可用，后端会回退 JSON。

风险：

- 并发写入可能互相覆盖。
- 大数据量性能较差。
- 不适合正式运营。

生产必须使用 MySQL。

### 2.3 缺少完整自动化测试

建议补充：

- API 单元测试。
- 上传安全测试。
- 订单身份隔离测试。
- 支付回调验签测试。
- 批量导入异常文件测试。
- 小程序核心流程冒烟测试。

### 2.4 管理后台是单页 HTML，后续可拆分模块

`cms/admin.html` 体积较大，功能多。

后续建议：

- 拆分 JS 模块。
