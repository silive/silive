## 八、备份检查与方案

### 当前权限不足的常见原因

此前 MySQL 备份容易失败，通常是以下原因之一：

1. `/www/backup/very-simple-custom` 目录归 root，普通 `ubuntu` 用户没有写权限。
2. `mysqldump -u root -p` 需要交互密码，自动部署脚本无法输入。
3. MySQL 开启权限限制，当前系统用户无法读取指定数据库。
4. 未使用 `.env` 中的 `MYSQL_USER/MYSQL_PASSWORD/MYSQL_DATABASE`，导致连接用户不正确。

### 已新增备份脚本

新增脚本：`scripts/production-backup.sh`

功能：

- 读取服务器项目目录 `.env`。
- 备份 MySQL 到 `/www/backup/very-simple-custom/mysql_时间.sql.gz`。
- 备份 `cms/uploads` 到 `/www/backup/very-simple-custom/uploads_时间.tar.gz`。
- 保留最近 14 天。
- 失败写入 `/www/backup/very-simple-custom/backup.log`。
- 不输出数据库密码。

建议首次在服务器执行：

```bash
sudo mkdir -p /www/backup/very-simple-custom
sudo chown -R ubuntu:ubuntu /www/backup/very-simple-custom
cd /www/wwwroot/very-simple-custom
bash scripts/production-backup.sh
```

如果必须使用 root MySQL 账号且 `.env` 里没有密码，建议在服务器创建只读备份账号，不要把 root 密码写进脚本。
